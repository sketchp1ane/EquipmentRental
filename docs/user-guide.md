## 1. 通用操作（所有已登录用户）

### 登录

- 访问 `/Account/Login`，输入用户名（邮箱格式）和密码
- 连续 5 次失败后账号锁定 30 分钟
- 勾选"记住我"可保持 7 天免登录

### 修改个人信息 / 密码

- 点击右上角用户名 → "个人信息"
- 可修改姓名、电话、邮箱
- "修改密码"需先输入旧密码

### 首页看板

登录后自动跳转首页，展示：
- 设备数量卡片（总数 / 出租中 / 空闲 / 维修中 / 待审核）
- 近 30 天新增租赁折线图
- 即将到期证件预警列表（≤30 天）
- 待处理事项快捷入口

### 站内消息

- 右上角铃铛图标显示未读消息数
- 点击查看消息详情及跳转链接

## 2. 系统管理员（Admin）

拥有所有模块的完整权限，另有专属的用户管理功能。

### 用户管理

入口：左侧导航 → "用户管理" (`/Users`)

| 操作 | 说明 |
|---|---|
| 新增用户 | 填写姓名、邮箱、密码，分配一个或多个角色 |
| 编辑用户 | 修改基本信息、调整角色 |
| 启用 / 停用 | 停用后该账号无法登录 |
| 重置密码 | 直接设置新密码（临时密码） |
| 筛选查询 | 按姓名、角色、状态筛选 |

> 系统管理员可代替任意角色执行所有操作，适合演示和测试。

## 3. 设备管理员（DeviceAdmin）

负责设备台账的全生命周期维护，包括入库、证件管理、审核、故障处理和退场评价。

### 3.1 设备台账

入口：左侧导航 → "设备台账" (`/Equipment`)

| 操作 | 路径 | 说明 |
|---|---|---|
| 设备入库 | `/Equipment/Create` | 填写设备信息、上传图片（最多 5 张，JPG/PNG，单张 ≤5 MB） |
| 查看列表 | `/Equipment` | 支持按编号、名称、分类、状态、所属公司筛选 |
| 查看详情 | `/Equipment/{id}` | 完整信息 + 历史租赁记录 |
| 编辑设备 | `/Equipment/{id}/Edit` | 修改设备基本信息 |
| 导出 Excel | 列表页"导出"按钮 | 导出当前筛选结果 |

### 3.2 证件管理

入口：设备详情页 → "证件管理" (`/Qualification/{equipmentId}`)

| 操作 | 说明 |
|---|---|
| 新增证件 | 选择证件类型，填写编号、签发机构、有效期，上传附件（PDF/图片） |
| 编辑证件 | 更新有效期或替换附件 |
| 到期预警 | ≤30 天到期的证件在首页自动高亮显示 |

证件类型：产品合格证、出厂检验报告、特种设备使用登记证、年度检验报告、保险凭证、安装资质证明

### 3.3 资质审核

入口：左侧导航 → "资质审核" (`/Audit`)

| 操作 | 说明 |
|---|---|
| 查看待审核列表 | 展示所有状态为"待审核"的设备 |
| 审核通过 | 设备状态变为"空闲"，通知提交人 |
| 驳回 | 需填写驳回原因，设备状态保持"待审核" |

> 可审核的角色：DeviceAdmin、Admin。只读审计员（Auditor）无审核权限，访问返回 403。

### 3.4 故障工单处理

入口：左侧导航 → "故障工单" (`/Fault`)

| 操作 | 说明 |
|---|---|
| 接受工单 | 将工单状态从"待处理"变为"处理中" |
| 关闭工单 | 填写处理结果、维修费用，关闭工单，设备状态恢复 |

### 3.5 退场评价

入口：退场申请详情页 → "填写评价"（`/Return/{id}/Evaluate`）

| 字段 | 说明 |
|---|---|
| 设备外观评分 | 1–5 分 |
| 功能完好性评分 | 1–5 分 |
| 损耗描述 | 文字说明 + 照片 |
| 损耗扣款 | 输入扣款金额，系统自动计算退还押金 |

提交后通知项目负责人，设备状态更新（空闲 / 维修中 / 已报废）。

## 4. 调度员（Dispatcher）

负责响应用车申请、排期调度、生成合同。

### 4.1 调度管理

入口：左侧导航 → "调度管理" (`/Dispatch`)

**处理用车申请流程：**

1. 查看"待处理"申请列表
2. 点击申请 → 进入排期页面（`/Dispatch/Order?requestId={id}`）
3. 选择符合条件的空闲设备（系统自动过滤证件过期、时间冲突的设备）
4. 确认实际用车日期、日租金、押金
5. 提交 → 生成调度单 + 合同草稿；设备仍为**空闲**，待进场核验通过后才转为**出租中**

### 4.2 调度日历

入口：调度管理 → "日历视图" (`/Dispatch/Calendar`)

- 横轴为日期，纵轴为设备
- 直观展示各设备占用情况，辅助排期

### 4.3 合同管理

入口：调度单详情页 → "查看合同" (`/Contract/{id}`)

| 操作 | 说明 |
|---|---|
| 在线预览 | HTML 渲染合同内容 |
| 导出 PDF | 生成标准合同 PDF 文件 |
| 上传扫描件 | 线下签署后，上传盖章扫描件（PDF/图片） |
| 更新合同状态 | 草稿 → 待签署 → 已签署 |

> **联动说明**：上传扫描件会在同一事务中把关联调度单从"待签署"推进为"已签署"；未完成扫描件上传前，订单详情页不会显示进场核验码区块。

## 5. 项目负责人（ProjectLead）

负责提交用车需求、确认进场核验、发起退场。

### 5.1 提交用车申请

入口：左侧导航 → "用车申请" (`/Dispatch/Request`)

填写字段：项目名称、项目地址、设备类型、数量、预计用车日期区间、特殊要求、联系人、联系电话。

### 5.2 进场核验

入口：调度单详情页 → "进场核验" 或直接访问 `/Verification/{orderId}`

**核验流程：**

1. 在系统中输入核验码（UUID），或通过扫码跳转
2. 系统自动校验：设备编号、证件有效期、设备状态、核验码有效性（租赁开始日起 3 天内有效）
3. 全部通过 → 核验成功，记录时间和操作人
4. 任一不通过 → 显示具体失败原因

> **核验通过后**：调度单状态推进为"进行中"，设备状态从"空闲"翻转为"出租中"。

### 5.3 查看合同

入口：调度单详情页 → "查看合同"（只读）

### 5.4 故障上报

入口：左侧导航 → "故障上报" (`/Fault/Create?orderId={id}`)

填写故障描述、严重程度（轻微 / 中等 / 严重）、上传现场图片。提交后设备管理员收到通知。

### 5.5 安全交底签署

入口：安全交底记录详情页 → "确认签署"

安全员创建交底记录并添加参与人后，项目负责人点击"确认签署"完成电子签名。

### 5.6 退场申请

入口：调度单详情页 → "申请退场" (`/Return/{orderId}/Apply`)

填写实际退场日期和设备总体状况描述，提交后等待设备管理员完成退场评价。

## 6. 安全员（SafetyOfficer）

负责安全交底记录的创建与管理，以及日常巡检和故障上报。

### 6.1 安全交底

入口：左侧导航 → "安全交底" (`/Safety/List`)

**创建交底记录（需已核验进场）：**

1. 进入对应调度单详情页 → "新建安全交底"（`/Safety/Create?orderId={id}`）
2. 填写：交底日期、地点、作业内容、安全注意事项（富文本）
3. 添加参与人员（姓名、工种、联系电话）
4. 上传附件（照片或 PDF，最多 10 个）
5. 提交保存（草稿状态）

**签署完成：**

- 安全员和项目负责人分别在详情页点击"确认签署"
- 所有必须签署方完成后，记录状态变为"已完成"，不可再修改
- 已完成的交底记录可导出 PDF

### 6.2 日常巡检

入口：左侧导航 → "巡检记录" → "新增巡检"（`/Inspection/Create?orderId={id}`）

填写：巡检日期、**8 项固定检查项**（整机外观 / 液压系统 / 电气系统 / 紧固件 / 安全装置 / 控制装置 / 作业环境 / 操作记录），每项支持 **正常 / 异常 / 不适用** 三态；整体状态（正常 / 异常）随各项聚合；可填备注、上传现场照片。

### 6.3 故障上报

入口：左侧导航 → "故障上报"（`/Fault/Create?orderId={id}`）

与项目负责人权限相同，可提交故障工单。

## 8. 完整业务流程示例

以下展示一次完整的设备租赁流程，涉及 5 个角色的协作：

```
[设备管理员] 设备入库登记（含图片、证件上传）
      ↓
[设备管理员] 审核通过 → 设备状态：空闲
      ↓
[项目负责人] 提交用车申请
      ↓
[调度员] 排期选备 → 生成调度单 + 合同草稿
      ↓
[调度员] 上传合同扫描件 → 合同状态：已签署
      ↓
[项目负责人] 进场核验（输入核验码）→ 核验通过
      ↓
[安全员] 创建安全交底记录 → 添加参与人
      ↓
[安全员 + 项目负责人] 分别电子签署 → 交底状态：已完成
      ↓
[安全员 / 项目负责人] 日常巡检 / 故障上报（如有）
      ↓
[设备管理员] 处理故障工单（如有）→ 设备状态恢复
      ↓
[项目负责人] 发起退场申请
      ↓
[设备管理员] 填写退场评价 → 计算押金退还 → 设备状态：空闲
```
