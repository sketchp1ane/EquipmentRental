## 发现的问题

### 1. 非设备角色仍显示 `设备台账` 菜单

- 复现账号：`demo.safetyofficer@equiprental.com`
- 现象：
  - 访问 `/Audit` 会正确跳转到 `/Account/AccessDenied?ReturnUrl=%2FAudit`
  - 但首页侧栏仍出现 `设备管理 -> 设备台账`
- 影响：
  - 与 [docs/demo-guide.md](../demo-guide.md) Step 1 中“安全员没有设备管理、调度管理等菜单”的演示说法不一致
  - 也会让角色边界在答辩演示时显得不够清晰
- 初步判断：
  - 更像是侧栏渲染条件过宽，而不是 Controller 鉴权失效

### 2. 故障关闭后的状态文案与业务链路描述不一致

- 复现链路：故障工单 `#2`
- 现象：
  - DeviceAdmin 关闭工单后，页面提示“设备已恢复空闲状态”
  - 但 [docs/demo-guide.md](../demo-guide.md) Step 7b 描述的是“设备状态自动恢复为出租中”
  - 在当前订单仍处于活跃链路时，这个表述会引起状态理解混乱
- 影响：
  - 演示口径与页面反馈不一致
  - 如果服务层也确实回写为空闲，则会与“进行中的订单”业务语义冲突
- 建议：
  - 先核对 `FaultService.CloseFaultAsync` 的真实状态回写
  - 若服务层正确，仅需修正文案
  - 若服务层也回写为空闲，需要补状态机修正
