# Admin 单账号答辩演示脚本

本文档用于答辩现场演示：只登录 `admin@equiprental.com`，从设备入库一路完成到退场评价，不切换账号、不手改数据库、不直接访问隐藏 URL。

已验证结果见：`docs/qa/admin-flow-retest-20260507.md`。

## 1. 演示目标

- 证明 Admin 账号具备完整业务链路演示能力。
- 展示系统覆盖建筑设备租赁全生命周期：

```text
设备入库 -> 资质审核 -> 用车申请 -> 调度合同 -> 进场核验 -> 安全交底 -> 巡检故障 -> 退场评价
```

- 建议时长：12-15 分钟。
- 演示口径：以业务闭环、权限设计、状态机流转和可追溯记录为主，不展开多角色负例测试。

## 3. 现场演示路线

### Step 1 登录与权限总览（约 1 分钟）

1. 登录 `admin@equiprental.com`。
2. 打开首页看板，展示设备状态、待办入口、证件预警和租赁趋势。
3. 快速扫过导航入口：用户管理、设备台账、资质审核、用车申请、调度、合同、核验、安全交底、巡检、故障、退场。

讲解重点：

- 系统通过 RBAC 区分多角色职责。
- 本次使用 Admin 单账号，是为了在答辩现场稳定展示完整业务闭环。

检查点：

- 页面显示 Admin 已登录。
- 关键模块入口可见。

### Step 2 设备入库与资质（约 2 分钟）

1. 进入设备台账，新建设备。
2. 填写设备编号、名称、分类、品牌型号、购置日期等基础信息。
3. 上传 `demo-equipment.png`。
4. 提交后确认设备状态为 `待审核`。
5. 进入证件或资质管理，为该设备添加产品合格证或检测报告。
6. 上传 `demo-qualification.pdf`。

讲解重点：

- 新设备默认进入待审核状态。
- 资质文件受控上传，后续审核和到期预警都依赖这部分数据。

检查点：

- 设备详情能看到新建设备。
- 资质记录与附件保存成功。

### Step 3 资质审核（约 1 分钟）

1. 进入资质审核列表。
2. 找到刚新建的设备，打开审核详情。
3. 查看设备信息和资质证书。
4. 填写审核备注，点击通过。
5. 回到设备列表或详情页，确认设备状态变为 `空闲`。

讲解重点：

- 未审核设备不能被调度。
- 审核通过后设备进入可租赁资源池。

检查点：

- 审核通过后设备状态为 `空闲`。

### Step 4 用车申请与调度合同（约 3 分钟）

1. 进入用车申请，提交 `Admin 单账号演示项目`。
2. 填写项目地址、设备分类、需求时间、联系人信息。
3. 进入调度管理，处理该申请。
4. 选择刚审核通过的设备。
5. 填写实际起止日期、单价 `1200`、押金 `8000`。
6. 提交后生成调度单和合同。
7. 打开合同详情，上传 `demo-contract-scan.pdf`。

讲解重点：

- 调度选设备时会过滤不可用设备、资质不合格设备和时间冲突设备。
- 合同上传扫描件后才推进订单状态，避免未签署订单进入进场环节。

检查点：

- 合同状态为 `已签署`。
- 调度单进入可核验阶段，并显示核验码。

### Step 5 进场核验（约 1 分钟）

1. 从调度单详情复制核验码。
2. 进入进场核验页面。
3. 输入核验码并提交。
4. 查看核验结果。

讲解重点：

- 核验码用于现场交付确认。
- 核验结果会形成可追溯记录。

检查点：

- 调度单状态进入 `使用中`。
- 设备状态变为 `使用中`。
- 核验记录可在详情中查看。

### Step 6 安全交底（约 2 分钟）

1. 创建安全交底，选择当前调度单。
2. 填写交底地点、交底内容和参与人。
3. 保存后进入详情。
4. 使用 Admin 完成签署确认。

讲解重点：

- 安全交底支持富文本内容和签署留痕。
- 交底记录可以作为现场安全责任追溯依据。

检查点：

- 安全交底状态为 `已完成`。

### Step 7 使用监管：巡检与故障（约 2 分钟）

1. 新增一次巡检记录，选择当前调度单，结果填写正常。
2. 上报一次轻微故障。
3. 进入故障详情，点击受理。
4. 填写处理说明和维修费用，关闭工单。

讲解重点：

- 故障上报后设备进入维修中。
- 活跃订单上的故障关闭后，设备恢复为使用中。

检查点：

- 巡检记录新增成功。
- 故障工单状态为 `已关闭`。
- 设备状态恢复为 `使用中`。

### Step 8 退场申请与评价（约 2 分钟）

1. 从调度单或退场入口提交退场申请。
2. 打开退场详情页，确认状态为 `待评价`。
3. 点击页面可见的 `填写退场评价` 按钮。
4. 填写：
   - 外观评分：`5`
   - 功能评分：`5`
   - 扣款：`0`
   - 退场后设备状态：`空闲`
   - 备注：`Admin 单账号演示完成退场评价，押金全额退还。`
5. 提交后返回退场详情页。

讲解重点：

- Admin 可从 UI 可见入口进入退场评价，不需要切换设备管理员账号。
- 评价提交会同时完成退场申请、订单结算和设备状态回写。

检查点：

- 退场申请状态为 `已完成`。
- 调度单状态为 `已完成`。
- 设备状态为 `空闲`。
- 退款金额为 `¥8,000.00`。

## 4. 风险与兜底

| 风险 | 处理方式 |
|---|---|
| 登录态异常或提交时出现用户外键错误 | 退出登录或清理 Cookie 后重新登录 Admin |
| 文件上传失败 | 换用提前准备的小体积 PNG/PDF |
| 设备编号重复 | 在编号后追加 `-02` 或当前分钟 |
| 时间不够 | 优先完整演示主线：入库、审核、调度合同、核验、退场评价 |
| 某页面找不到刚创建记录 | 使用项目名或设备编号搜索 |

## 5. 最终验收标准

- 全程只使用 `admin@equiprental.com`。
- 不切换账号。
- 不直接访问隐藏 URL。
- 不手改数据库。
- 每个关键状态都通过页面可见信息确认。
- 最终能从退场详情页点击 `填写退场评价`，提交后完成退款、订单完结和设备归库。

## 6. 答辩收尾话术

> 通过这条链路可以看到，系统不是单点功能堆叠，而是围绕设备租赁生命周期建立了完整状态机。设备从入库、审核、调度、签约、进场、安全交底、巡检故障，到最终退场评价，每一步都有角色权限、业务状态和操作记录约束。为了答辩演示的连续性，本次用 Admin 单账号完成全流程；在真实使用中，各环节也可以按项目负责人、调度员、安全员、设备管理员等角色拆分执行。
