# 2 相关技术介绍

## 2.1 B/S 架构与 MVC 设计模式

B/S 架构即浏览器/服务器架构，用户通过浏览器访问系统，业务处理和数据存储主要由服务器完成。对于建筑租赁设备管理平台而言，B/S 架构能够降低客户端部署成本，便于多角色用户在不同终端访问系统。项目负责人、调度员、设备管理员和安全员只需要通过浏览器登录系统，即可完成用车申请、设备调度、安全交底、故障上报和退场评价等操作。

MVC 是一种常见的软件设计模式，将系统划分为模型、视图和控制器三个部分。模型用于表示数据和业务对象，视图负责页面展示，控制器负责接收请求并协调业务处理。Fowler 对企业应用分层架构和业务逻辑组织方式进行了系统阐述，分层设计有助于降低模块耦合，提高系统可维护性[10]。本系统在 MVC 基础上进一步引入 Service 层，将复杂业务规则和状态流转集中处理，避免控制器承担过多业务逻辑。

## 2.2 ASP.NET Core MVC 技术

ASP.NET Core MVC 是 Microsoft 提供的跨平台 Web 应用开发框架，支持路由、中间件、模型绑定、过滤器、依赖注入和 Razor 页面渲染等能力[11]。本系统采用 ASP.NET Core MVC 构建服务端渲染的 B/S 管理系统。用户请求首先进入 Controller，Controller 完成身份验证、角色授权和参数校验后调用业务服务，再将处理结果返回到 Razor View 或重定向到相应页面。

ASP.NET Core 内置依赖注入机制，能够将业务服务统一注册并注入到控制器中。本系统将 EquipmentService、DispatchService、SafetyService、ReturnService、NotificationService、FileService 等业务服务注册为作用域服务，使各模块之间职责边界更加清晰。该方式有利于后续维护和测试，也符合软件工程中低耦合、高内聚的设计原则。

## 2.3 EF Core 与 SQL Server 数据库

Entity Framework Core 是 Microsoft 提供的对象关系映射框架，能够将 C# 实体类与关系数据库表建立映射关系，支持 LINQ 查询、关系配置、迁移和事务处理等功能[12]。本系统采用 EF Core 的 Code First 方式进行数据库建模，通过实体类描述设备、证件、审核记录、调度单、合同、进场核验、安全交底、巡检记录、故障工单和退场评价等业务对象。

SQL Server 是 Microsoft 提供的关系型数据库管理系统，支持事务、索引、约束、视图和安全控制等能力[13]。本系统使用 SQL Server 2022 作为数据存储平台，保存用户信息、设备台账、资质证件、合同记录、安全记录、通知消息和审计日志等数据。通过 EF Core 与 SQL Server 的结合，系统能够在面向对象开发方式和关系数据库存储之间建立较为稳定的数据访问层。

## 2.4 ASP.NET Core Identity 与 RBAC 权限控制

ASP.NET Core Identity 是 ASP.NET Core 提供的用户认证和身份管理框架，支持用户登录、注销、密码管理、角色管理和安全令牌等功能[14]。本系统使用 Identity 管理用户账户，并结合角色授权实现不同用户的访问控制。系统定义系统管理员、设备管理员、调度员、项目负责人、安全员和只读审计员六类角色，不同角色拥有不同业务权限。

RBAC 即基于角色的访问控制。Sandhu 等提出的 RBAC 模型通过角色连接用户和权限，使权限分配更加清晰和可维护[15]。在本系统中，设备管理员负责设备台账、资质审核、故障处理和退场评价，调度员负责用车申请处理、设备排期和合同管理，项目负责人负责提交用车申请、进场核验和发起退场申请，安全员负责安全交底、巡检和故障上报，只读审计员只能查看记录而不能执行修改操作。系统还使用 BCrypt 对密码进行哈希处理，BCrypt 的自适应计算成本设计能够提高密码存储安全性[16]。

## 2.5 前端展示、文档导出与安全辅助技术

系统前端采用 Razor Views 进行页面渲染，结合 Bootstrap、jQuery 和 Chart.js 完成页面布局、交互和图表展示。Bootstrap 提供响应式布局和常用界面组件[20]，Chart.js 支持在浏览器中绘制图表[21]。本系统首页看板使用 Chart.js 展示租赁趋势，设备台账、调度、合同、安全交底等页面通过 Bootstrap 提供统一的管理界面风格。

在文档导出方面，系统使用 QuestPDF 生成合同和安全交底 PDF 文件，使用 EPPlus 导出设备台账 Excel 文件[22-23]。在安全防护方面，系统启用全局 CSRF 防护，对修改类请求进行防伪令牌校验。OWASP 对 CSRF、文件上传和 XSS 防护均给出了实践建议[17-19]。本系统结合文件扩展名白名单、MIME 类型校验、文件魔数校验、GUID 重命名和受控下载等方式降低上传文件风险；安全交底富文本内容在保存前通过 HtmlSanitizer 过滤危险 HTML，降低 XSS 风险。

## 2.6 本章小结

本章介绍了系统开发中使用的主要技术，包括 B/S 架构、MVC 设计模式、ASP.NET Core MVC、EF Core、SQL Server、ASP.NET Core Identity、RBAC、前端展示技术、文档导出组件和安全辅助技术。这些技术共同支撑了系统的页面展示、业务处理、数据持久化、权限控制、文档导出和安全防护，为后续系统设计与实现奠定了技术基础。
