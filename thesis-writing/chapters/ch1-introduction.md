# 1 绪论

## 1.1 研究背景与意义

建筑施工过程中需要使用大量机械设备和专业施工设备，例如起重设备、运输设备、高空作业设备、混凝土设备和辅助施工设备等。随着建筑项目规模扩大和专业化程度提高，施工企业或项目部往往不会完全自购所有设备，而是通过租赁方式获得设备资源。设备租赁能够降低一次性资产投入，提高设备利用率，但也带来了设备台账、资质证件、租赁调度、合同流转、进场核验和使用监管等方面的管理压力。

在传统管理模式下，设备租赁管理常依赖纸质台账、Excel 表格和线下沟通完成。该方式在小规模项目中具有一定可行性，但随着设备数量、项目数量和参与角色增加，容易出现台账信息更新不及时、设备证件有效期难以及时发现、合同签署过程难追踪、设备进场核验依赖人工比对、安全交底和巡检记录分散、故障维修责任不清等问题。建筑机械的安全使用、施工现场机械设备检查和建筑施工安全检查在行业规范中均有明确要求[1-3]，特种设备安全和建筑起重机械监管也受到相关法律法规约束[4-5]。因此，面向建筑租赁设备业务构建信息化、流程化和可追溯的管理平台具有现实意义。

本课题设计并实现的建筑租赁设备全生命周期管理平台，围绕“设备入库、资质审核、线上调度、进场核验、安全交底、使用监管、退场评价”构建完整业务闭环。系统不仅保存设备基础信息，而且通过状态字段反映设备、用车申请、调度单、合同、故障工单和退场申请的业务阶段。与传统台账式管理相比，该平台能够更好地支持多角色协同、流程约束和责任追溯，有助于提高设备租赁管理效率和安全管理水平。

## 1.2 国内外研究现状

从国内相关实践来看，建筑施工设备管理逐渐从人工记录、纸质台账和单机表格管理向信息系统管理转变。施工现场机械设备检查、建筑机械安全使用、建筑施工安全检查等规范为设备使用管理提供了制度依据[1-3]。同时，《中华人民共和国特种设备安全法》和《建筑起重机械安全监督管理规定》对特种设备和建筑起重机械的使用登记、安全监督和责任主体提出了要求[4-5]。这些制度和规范说明，建筑设备管理并非单纯资产登记问题，而是涉及资质、状态、责任和过程记录的综合管理问题。

从软件工程角度看，管理系统开发需要经过需求分析、系统设计、实现和测试等环节。ISO/IEC/IEEE 29148:2018 对需求工程过程进行了规范，强调系统需求应从利益相关者、业务目标和约束条件出发进行分析[6]；ISO/IEC 25010:2011 则从功能适合性、可靠性、易用性、安全性、可维护性等方面提出软件质量模型[7]。对于本课题而言，系统需求不仅包括设备台账、调度合同等功能需求，还包括权限隔离、数据一致性、业务可追溯性和文件上传安全等非功能需求。

国外软件工程研究和企业应用架构实践较早强调分层设计、职责分离和模型驱动的软件开发方法。Sommerville 和 Pressman 等学者在软件工程教材中均指出，需求分析、架构设计、编码实现和测试验证是软件项目质量保障的重要环节[8-9]。Fowler 在企业应用架构模式中对分层架构、领域逻辑组织和数据访问模式进行了系统阐述[10]。本系统采用 ASP.NET Core MVC 分层架构，将请求处理、业务规则、数据实体、页面模型和页面展示进行职责划分，符合管理系统模块清晰、易维护、可扩展的设计需求。

综上，现有规范和研究为建筑设备管理、需求分析、系统设计和软件质量评价提供了理论和方法支撑，但针对建筑租赁设备从入库到退场的完整业务闭环，仍需要结合具体场景进行系统化设计。因此，本文以建筑租赁设备管理为对象，设计并实现一个覆盖设备全生命周期的 Web 管理平台。

## 1.3 本文主要研究内容

本文围绕建筑租赁设备全生命周期管理需求，完成以下主要工作。

第一，分析建筑租赁设备管理中的业务痛点和用户角色，明确系统应支持的功能需求与非功能需求。系统角色包括系统管理员、设备管理员、调度员、项目负责人、安全员和只读审计员，不同角色承担不同业务职责。

第二，设计系统总体架构和功能模块。系统采用 B/S 架构和 ASP.NET Core MVC 分层模式，Controller 层负责请求处理和权限控制，Service 层负责业务规则和状态流转，Entity 层描述数据库实体，ViewModel 层承担页面输入输出，Razor View 层负责界面展示。

第三，设计数据库模型。系统以设备为核心实体，以调度单串联合同、进场核验、安全交底、巡检、故障处理和退场评价等过程数据，通过状态枚举和关联关系记录设备租赁全过程。

第四，实现系统核心功能并进行测试。系统实现设备台账、资质审核、线上调度、合同管理、进场核验、安全交底、巡检监管、故障处理、退场评价、首页看板、站内消息和文件安全访问等模块，并从功能、权限、流程和安全等方面进行验证。

## 1.4 论文组织结构

本文共分为八章。第 1 章为绪论，介绍研究背景、研究现状、研究内容和论文结构。第 2 章介绍系统开发中使用的相关技术。第 3 章从可行性、用户角色、功能需求、非功能需求和业务流程等方面进行需求分析。第 4 章阐述系统总体设计，包括架构设计、模块设计、权限设计和状态流转设计。第 5 章介绍数据库设计，包括实体关系、核心表结构、状态枚举和约束设计。第 6 章说明系统详细设计与实现，重点介绍各核心业务模块的实现过程。第 7 章进行系统测试，包括功能测试、权限测试、业务流程测试和安全性测试。第 8 章总结全文工作，并提出后续改进方向。
