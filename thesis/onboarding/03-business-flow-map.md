# 03 业务流程地图

这份文件把系统当成一条业务链来看。阅读时重点关注：谁操作、输入什么、产生什么记录、状态怎么变。

## 总流程

```text
设备入库 -> 资质审核 -> 线上调度 -> 合同签署 -> 进场核验
        -> 安全交底 -> 使用监管 -> 退场申请 -> 退场评价
```

## 阶段拆解

| 阶段 | 操作角色 | 主要输入 | 主要输出 | 状态变化 |
|---|---|---|---|---|
| 设备入库 | 设备管理员/管理员 | 设备编号、分类、品牌型号、图片、所属公司 | `Equipment`、`EquipmentImages` | 设备进入 `PendingReview` |
| 证件维护 | 设备管理员/管理员 | 合格证、年检报告、保险等证件 | `Qualifications` | 为审核和调度提供资质依据 |
| 资质审核 | 设备管理员/管理员 | 审核动作、备注 | `AuditRecords` | 通过后设备 `PendingReview -> Idle` |
| 用车申请 | 项目负责人/管理员 | 项目、地址、设备类型、用车日期、联系人 | `DispatchRequests` | 申请进入 `Pending` |
| 调度排期 | 调度员/管理员 | 设备、实际租期、日租金、押金 | `DispatchOrders`、`Contracts` | 申请 `Scheduled`，调度单 `Unsigned` |
| 合同签署 | 调度员/管理员 | 上传线下签署扫描件 | 合同扫描件 | 合同 `Signed`，调度单 `Signed`，设备 `InUse` |
| 进场核验 | 项目负责人/管理员 | 核验码 | `EntryVerifications` | 核验通过后调度单 `InProgress`，设备保持 `InUse` |
| 安全交底 | 安全员/管理员 | 交底内容、参与人、附件 | `SafetyBriefings`、参与人、附件 | 参与方签署完成后 `Completed` |
| 日常巡检 | 安全员/管理员 | 固定检查项、整体状态、备注、照片 | `InspectionRecords` | 记录使用过程检查结果 |
| 故障处理 | 安全员/项目负责人/设备管理员/管理员 | 故障描述、维修结果 | `FaultReports` | 故障可使设备进入 `Maintenance`，关闭后恢复 |
| 退场申请 | 项目负责人/管理员 | 实际退场日期、设备状况 | `ReturnApplications` | 进入 `PendingEvaluation` |
| 退场评价 | 设备管理员/管理员 | 评分、扣款、设备后续状态 | `ReturnEvaluations` | 调度单 `Complete`，设备回到空闲/维修/报废等状态 |

## 三条核心状态线

### 设备状态线

```text
PendingReview -> Idle -> InUse -> Idle
                      -> Maintenance -> Idle
                      -> Scrapped
```

答辩重点：设备不是简单 CRUD，而是被业务流程驱动状态变化。

### 调度单状态线

```text
Unsigned -> Signed -> InProgress -> Complete
                         -> Terminated
```

答辩重点：未上传合同扫描件时，调度单不能进场核验；上传扫描件时设备已进入 `InUse`，进场核验主要推进调度单和生成核验记录。

### 安全与故障线

```text
SafetyBriefing: Draft -> Completed
FaultReport: Pending -> InProgress -> Closed
```

答辩重点：安全交底和故障工单把现场行为变成可追溯记录。

## 演示时的讲述顺序

1. 先说“系统围绕设备生命周期设计，不是孤立页面集合”。
2. 展示设备详情，让评委看到设备、证件、图片、租赁历史都集中在台账里。
3. 展示调度和合同，说明租赁业务从申请到合同可追踪。
4. 展示核验码，说明线下进场变成线上校验。
5. 展示安全交底、故障和退场评价，说明系统覆盖使用中和结束后的管理。

## 容易混淆的点

| 容易混淆 | 正确理解 |
|---|---|
| 审计员是不是审核员 | 不是。只读审计员只能查看，不能执行资质审核 |
| 合同签署是不是电子签章 | 不是。本期是线下签署后上传扫描件 |
| 设备空闲是否一定能调度 | 不一定。还要检查资质有效期和时间冲突 |
| 故障关闭后一定回空闲吗 | 要结合订单和维修结果，设备状态由服务层规则处理 |
| Controller 是否处理业务规则 | 不应该。业务规则放在 `Services/` |
