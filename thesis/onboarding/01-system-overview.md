# 01 系统总览

## 一句话定位

EquipmentRental 是一个建筑租赁设备全生命周期管理系统，用 B/S 架构把设备从入库登记到退场评价的流程串起来，解决设备状态不透明、合同流转慢、安全记录难追溯、退场结算不规范等问题。

## 业务背景

传统建筑设备租赁管理常用纸质台账或 Excel，主要问题是：

| 痛点 | 系统对应能力 |
|---|---|
| 设备状态更新不及时 | 设备台账和状态流转 |
| 资质证件容易过期 | 证件管理、到期预警、调度前校验 |
| 合同线下流转难追踪 | 调度单、合同草稿、PDF 导出、扫描件上传 |
| 进场靠人工核对 | 唯一核验码和二维码 |
| 安全交底记录分散 | 富文本交底、参与人签署、PDF 导出 |
| 维修和退场责任不清 | 巡检、故障工单、退场评价、押金计算 |

## 六个角色

| 角色 | 主要职责 | 答辩时怎么讲 |
|---|---|---|
| 系统管理员 | 用户管理、全局维护 | 最高权限，负责平台运维 |
| 设备管理员 | 设备入库、资质审核、故障处理、退场评价 | 代表租赁公司管理设备资产 |
| 调度员 | 用车申请处理、设备排期、合同生成 | 负责把空闲设备安排给项目 |
| 项目负责人 | 提交申请、进场核验、退场申请 | 代表施工项目使用设备 |
| 安全员 | 安全交底、巡检、故障上报 | 负责现场安全和风险记录 |
| 只读审计员 | 跨模块查看历史记录 | 只能查看，不能修改，体现审计隔离 |

## 七阶段主流程

```text
设备入库
  -> 资质审核
  -> 线上调度
  -> 进场核验
  -> 安全交底
  -> 使用监管
  -> 退场评价
```

每个阶段都有明确的角色、状态和数据记录。答辩时不要把它讲成“很多管理页面”，要讲成“一条设备租赁业务链路”。

## 技术栈快照

| 层面 | 技术 |
|---|---|
| 后端 | C# 13 / .NET 10 / ASP.NET Core MVC |
| 数据访问 | EF Core 10 / SQL Server 2022 |
| 认证授权 | ASP.NET Core Identity + RBAC + BCrypt |
| 前端 | Razor Views / Bootstrap 5 / jQuery / Chart.js |
| 文档导出 | QuestPDF 导出 PDF，EPPlus 导出 Excel |
| 安全 | 全局 CSRF、HtmlSanitizer、文件上传白名单和魔数校验 |

## 你最应该先记住的状态

| 对象 | 状态 |
|---|---|
| 设备 | 待审核、空闲、出租中、维修中、已报废 |
| 用车申请 | 待处理、已排期、已取消 |
| 调度单 | 待签署、已签署、进行中、已完成、已终止 |
| 合同 | 草稿、待签署（枚举保留，主流程未写入）、已签署、已终止 |
| 安全交底 | 草稿、已完成 |
| 故障工单 | 待处理、处理中、已关闭 |
| 退场申请 | 待评价、已完成 |
