# 02 启动和演示

## 本地启动

项目默认使用 SQL Server 2022 Docker 容器，应用启动时会自动执行迁移并写入演示数据。

```bash
cd /Users/sketchplane/EquipmentRental

# 如果数据库容器已经创建过
docker start equiprental-db

# 首次或需要同步数据库时
dotnet ef database update

# 启动应用
dotnet run
```

优先访问：

```text
http://localhost:5085
```

说明：`Properties/launchSettings.json` 当前配置的本地 HTTP 地址是 `http://localhost:5085`。如果终端输出了其他监听地址，以终端输出为准。

## 15 分钟演示路线

这条路线适合答辩现场，重点展示“业务闭环”和“权限分工”。

| 步骤 | 登录角色 | 页面/操作 | 要讲的点 |
|---|---|---|---|
| 1 | 管理员 | 用户管理 | 系统支持 6 类角色，按角色授权 |
| 2 | 设备管理员 | 设备台账、设备详情 | 设备有状态、图片、证件和租赁历史 |
| 3 | 设备管理员 | 新建设备和证件 | 新设备默认待审核，不能直接调度 |
| 4 | 设备管理员 | 资质审核 | 审核通过后设备进入空闲状态 |
| 5 | 项目负责人 | 提交用车申请 | 项目侧发起租赁需求 |
| 6 | 调度员 | 排期并生成合同 | 系统过滤可用设备并生成调度单/合同 |
| 7 | 调度员 | 上传合同扫描件 | 合同、调度单进入已签署，设备进入使用中 |
| 8 | 项目负责人 | 进场核验 | 核验码通过后调度单进入进行中并生成核验记录 |
| 9 | 安全员 | 安全交底/巡检/故障上报 | 留存现场安全和风险记录 |
| 10 | 项目负责人/设备管理员 | 退场申请/评价 | 完成退场、押金和设备状态更新 |

## 答辩前检查清单

- 数据库容器 `equiprental-db` 能启动。
- `dotnet run` 无报错，浏览器能打开 `http://localhost:5085`。
- 至少提前登录过管理员、设备管理员、调度员、项目负责人、安全员 5 个账号。
- 准备一张设备图片和一个 PDF 文件，用于演示上传。
- 打开 [../../docs/demo-guide.md](../../docs/demo-guide.md) 作为完整演示脚本备用。
- 如果现场时间不足，优先演示：登录权限、设备台账、调度合同、核验码、安全交底、退场评价。

## 常见启动问题

| 问题 | 排查方式 |
|---|---|
| 数据库连不上 | 确认 Docker 已启动，容器名是 `equiprental-db`，1433 端口未被占用 |
| 访问地址不对 | 看 `dotnet run` 终端输出；当前 launchSettings 默认是 `5085` |
| 登录失败 | 检查是否使用演示账号密码，注意大小写 |
| 页面样式异常 | 确认 `wwwroot/lib` 静态资源存在，浏览器控制台无 404 |
