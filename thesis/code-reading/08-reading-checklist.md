# 08 源码阅读检查清单

用这份清单检查自己是否真正看懂项目。能回答大部分问题，就说明你已经能比较稳地讲代码了。

## 全局入口

- [ ] 能说明 `Program.cs` 里注册了哪些关键能力：EF Core、Identity、MVC、Services、HtmlSanitizer。
- [ ] 能说出默认路由为什么会进入 `HomeController.Index`。
- [ ] 能解释启动时为什么会自动迁移数据库并写入种子数据。
- [ ] 能找到当前本地启动地址配置在哪里。

## 分层理解

- [ ] 能说清 Controller 和 Service 的职责区别。
- [ ] 能通过一个页面找到对应 ViewModel。
- [ ] 能从一个 POST Action 跳到真正执行业务规则的 Service 方法。
- [ ] 能说明 Entity 和 ViewModel 为什么不混用。

## 数据模型

- [ ] 能画出 `Equipment -> DispatchOrder -> Contract / EntryVerification / ReturnApplication` 的关系。
- [ ] 能说出设备、调度单、合同、故障、退场申请的主要状态。
- [ ] 能找到唯一约束：设备编号、核验码、合同订单关系、退场申请订单关系。
- [ ] 能解释为什么 `Uploads/` 不放在 `wwwroot/`。

## 业务流程

- [ ] 能顺着代码讲出“设备入库 -> 资质审核 -> 空闲”的流程。
- [ ] 能顺着代码讲出“用车申请 -> 调度单 -> 合同 -> 核验”的流程。
- [ ] 能顺着代码讲出“故障上报 -> 受理 -> 关闭 -> 设备状态恢复”的流程。
- [ ] 能顺着代码讲出“退场申请 -> 评价 -> 押金退款 -> 订单完成”的流程。

## 权限安全

- [ ] 能找到所有角色常量。
- [ ] 能解释 `[Authorize]` 在类和 Action 上的区别。
- [ ] 能说明全局 CSRF 验证在哪里启用。
- [ ] 能说出 `FileService` 的五类上传校验。
- [ ] 能说明富文本为什么要经过 HtmlSanitizer。

## 常见修改定位

- [ ] 想改一个表单字段时，知道先看 ViewModel 和 View。
- [ ] 想改一个状态流转时，知道先看 Service。
- [ ] 想改一个角色权限时，知道先看 Controller 和侧边栏。
- [ ] 想改一个数据库字段时，知道要改 Entity、DbContext、迁移和相关页面。
- [ ] 想改演示数据时，知道看 `Data/DbInitializer.cs`。

## 最终自测题

1. 为什么审核通过之前的设备不能被调度？
2. 为什么合同扫描件上传会影响调度单状态？
3. 进场核验失败时系统会不会写记录？
4. 故障关闭后设备为什么有时回到 `InUse`，有时回到 `Idle`？
5. 退场评价里的扣款金额为什么不能超过押金？
6. 如果要允许上传 Word 文档，需要改哪些地方？
7. 如果要给审计员开放某个列表但不开放新增按钮，需要改哪些地方？
