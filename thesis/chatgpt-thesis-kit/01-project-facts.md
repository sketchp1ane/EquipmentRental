# 01 项目事实清单

本文用于统一论文中的项目事实。写作时优先使用这里的口径。

## 项目定位

本项目是一个基于 ASP.NET Core MVC 的建筑租赁设备全生命周期管理平台，用于管理建筑设备从入库登记、资质审核、调度租赁、合同流转、进场核验、安全交底、使用监管到退场评价的完整业务流程。

## 业务痛点

- 设备台账依赖纸质或 Excel，状态更新不及时。
- 设备证件和年检信息分散，过期风险难以及时发现。
- 用车申请、排期调度和合同流转依赖线下沟通。
- 设备进场核验依赖人工比对，效率低且容易出错。
- 安全交底、巡检和故障处理记录缺少统一追溯。
- 退场评价和押金扣款缺乏标准化记录。

## 技术事实

| 项目 | 当前实现 |
|---|---|
| 目标框架 | `.NET 10`，`net10.0` |
| 后端框架 | ASP.NET Core MVC |
| 语言 | C# 13 |
| ORM | EF Core 10 |
| 数据库 | SQL Server 2022 |
| 认证授权 | ASP.NET Core Identity + 角色授权 |
| 密码哈希 | BCrypt，替换 Identity 内置默认哈希实现 |
| 前端 | Razor Views + Bootstrap 5 + jQuery 3 + Chart.js 4 |
| PDF 导出 | QuestPDF |
| Excel 导出 | EPPlus |
| 二维码/核验码 | QRCoder + GUID 核验码 |
| 富文本过滤 | HtmlSanitizer |
| 本地默认地址 | `http://localhost:5085` |

## 用户角色

| 角色常量 | 中文角色 | 论文描述 |
|---|---|---|
| `Admin` | 系统管理员 | 负责用户管理和全局运维 |
| `DeviceAdmin` | 设备管理员 | 负责设备台账、证件、审核、故障和退场评价 |
| `Dispatcher` | 调度员 | 负责用车申请处理、设备排期和合同管理 |
| `ProjectLead` | 项目负责人 | 负责提交用车申请、进场核验和退场申请 |
| `SafetyOfficer` | 安全员 | 负责安全交底、巡检和故障上报 |
| `Auditor` | 只读审计员 | 负责只读查看历史记录和验证权限隔离 |

## 核心模块

| 模块 | 主要能力 |
|---|---|
| 用户与认证 | 登录、注销、个人信息、密码修改、用户管理 |
| 设备台账 | 设备分类、设备入库、图片上传、列表筛选、详情、Excel 导出 |
| 资质审核 | 证件管理、到期预警、审核通过、审核驳回 |
| 线上调度 | 用车申请、可用设备筛选、调度排期、调度日历 |
| 合同管理 | 合同草稿、在线预览、PDF 导出、扫描件上传 |
| 进场核验 | 核验码展示、核验结果记录、状态推进 |
| 安全交底 | 富文本交底、参与人签署、附件、PDF 导出 |
| 使用监管 | 巡检记录、固定巡检项、现场照片 |
| 故障处理 | 故障上报、工单接受、维修处理、关闭恢复 |
| 退场评价 | 退场申请、评分、损耗扣款、押金退还、设备状态更新 |
| 首页看板 | 设备统计、租赁趋势、证件预警、角色待办 |
| 站内消息 | 通知提醒、未读消息、跳转链接 |
