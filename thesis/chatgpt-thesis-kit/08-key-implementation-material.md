# 08 关键功能实现写作素材

## 用户认证与权限控制

实现要点：

- 使用 ASP.NET Core Identity 处理登录、注销、角色和账号锁定。
- 使用 BCrypt 替换默认密码哈希。
- 使用 `[Authorize]` 和角色常量控制页面和操作权限。
- 用户管理模块支持新增、编辑、启停和重置密码。

论文写法：

用户登录后，系统根据其所属角色展示不同菜单并限制操作权限。未授权用户访问受保护页面时会被重定向或返回拒绝访问页面，从而实现管理系统中的权限隔离。

## 设备台账与资质审核

实现要点：

- 设备入库保存设备基础信息和图片。
- 新建设备默认进入待审核状态。
- 证件模块维护合格证、年检报告、保险等资质信息。
- 资质审核通过后设备进入空闲状态，驳回时记录原因。
- 首页展示即将到期证件预警。

论文写法：

设备台账模块不仅保存设备基础资料，还通过状态字段约束设备是否能够进入后续租赁流程。资质审核模块将设备证件信息和审核结果关联起来，保证未通过审核的设备不能直接参与调度。

## 线上调度与合同管理

实现要点：

- 项目负责人提交用车申请。
- 调度员根据设备类型、租期和资质筛选可用设备。
- 调度排期后生成调度单和合同草稿。
- 合同支持在线预览和 PDF 导出。
- 上传合同扫描件后，合同和调度单进入已签署状态，设备同步进入使用中。

论文写法：

调度模块是连接项目需求与设备资源的核心环节。系统在生成调度单前校验设备状态、资质有效期和时间冲突，避免不可用设备被分配。合同扫描件上传后，系统同步推进合同、调度单和设备状态，为进场核验提供前置条件。

## 进场核验

实现要点：

- 调度单签署后生成或展示核验码。
- 项目负责人输入核验码进行进场核验。
- 服务层检查调度单状态、核验码、设备状态和证件有效期。
- 核验通过后生成核验记录，调度单进入进行中。
- 设备状态不由核验步骤改变；当前主流程在合同扫描件上传时已将设备置为出租中。
- 核验失败也保留失败原因。

论文写法：

进场核验模块将传统人工比对流程转化为系统校验流程。系统通过核验码与调度单关联，确保只有已签署合同的设备才能进入现场使用阶段，提高核验效率和流程可追溯性。

## 安全交底与使用监管

实现要点：

- 安全员创建安全交底记录。
- 支持富文本内容、参与人、附件和签署确认。
- 富文本入库前经过 HtmlSanitizer 过滤。
- 巡检模块记录固定检查项、整体状态、备注和现场图片。
- 故障模块支持上报、接受、处理和关闭。

论文写法：

安全交底模块用于沉淀设备进场后的安全管理资料，参与人签署信息能够记录签署时间和相关账号。巡检和故障模块将设备使用过程中的现场情况转化为结构化记录，为后续责任追溯和退场评价提供依据。

## 退场评价

实现要点：

- 项目负责人发起退场申请。
- 设备管理员填写退场评价。
- 评价包含外观评分、功能评分、损耗描述和损耗扣款。
- 系统根据押金和扣款计算退还金额。
- 提交后更新调度单和设备后续状态。

论文写法：

退场评价模块是设备租赁流程的收尾环节。系统通过评分、损耗记录和扣款金额对设备使用结果进行量化，并将评价结果与调度单和设备状态关联，形成完整的租赁闭环。

## 首页看板与通知

实现要点：

- 首页展示设备总数、出租中、空闲、维修中和待审核数量。
- 展示租赁趋势和证件到期预警。
- 根据角色展示待办入口。
- 站内消息提醒审核、交底、退场等业务节点。

论文写法：

首页看板通过聚合设备状态、租赁趋势和待办事项，为不同角色提供入口和业务提醒。站内消息模块进一步提高跨角色协作效率。
