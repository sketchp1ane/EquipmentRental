# 10 测试章节写作素材

## 测试目标

系统测试目标是验证各业务模块功能是否符合需求，角色权限是否隔离，状态流转是否正确，文件上传和富文本等安全机制是否有效。

## 测试环境

可按实际情况写：

| 项目 | 内容 |
|---|---|
| 操作系统 | macOS 或实际测试环境 |
| 后端运行环境 | .NET 10 SDK |
| 数据库 | SQL Server 2022 Docker 容器 `equiprental-db` |
| 浏览器 | Chrome 或 Edge |
| 应用地址 | `http://localhost:5085` |

## 业务流程测试

主流程：

```text
设备入库 -> 添加证件 -> 审核通过 -> 提交用车申请 -> 调度排期
-> 上传合同扫描件 -> 进场核验 -> 安全交底 -> 巡检/故障
-> 退场申请 -> 退场评价
```

验证重点：

- 每一步状态是否正确推进。
- 不满足前置条件时是否被阻止。
- 不同角色是否只能执行自己的操作。
- 业务结果是否能在详情页和看板中体现。

## 安全测试

| 测试点 | 预期结果 |
|---|---|
| 上传不允许的文件类型 | 系统拒绝上传 |
| 上传伪造扩展名文件 | 魔数校验失败 |
| 提交缺少防伪令牌的 POST 请求 | 请求被拒绝 |
| 富文本中插入危险脚本 | 保存前被过滤 |
| 直接访问上传目录外路径 | 被路径安全检查阻止 |
