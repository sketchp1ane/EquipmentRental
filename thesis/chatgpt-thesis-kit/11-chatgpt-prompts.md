# 11 ChatGPT 提示词模板

复制提示词前，建议先给 ChatGPT `00-chatgpt-context-pack.md` 和 `12-consistency-rules.md`。

## 生成论文目录

```text
你现在是一名本科毕业论文写作助手。请基于我提供的 EquipmentRental 项目上下文，为题目“基于 ASP.NET Core MVC 的建筑租赁设备全生命周期管理平台的设计与实现”生成一份中文毕业论文目录。

要求：
1. 目录符合软件工程类本科毕业论文结构。
2. 包含绪论、相关技术、需求分析、总体设计、数据库设计、详细设计与实现、系统测试、总结与展望。
3. 不要写项目未实现的功能。
4. 每个二级标题后用一句话说明该节写作重点。
```

## 写摘要

```text
请根据我提供的项目上下文，写一段中文毕业论文摘要，题目为“基于 ASP.NET Core MVC 的建筑租赁设备全生命周期管理平台的设计与实现”。

要求：
1. 字数约 400 字。
2. 包含研究背景、系统目标、关键技术、主要功能和测试结果。
3. 语言正式但不要空泛。
4. 关键词给出 4 到 6 个。
5. 不要编造 GPS、移动端 App、ERP 对接或在线电子签章等未实现功能。
```

## 写需求分析章节

```text
请基于项目上下文写“需求分析”章节。

要求：
1. 包含业务需求分析、用户角色分析、功能性需求、非功能性需求和业务流程分析。
2. 角色包括系统管理员、设备管理员、调度员、项目负责人、安全员、只读审计员。
3. 核心流程必须写成“设备入库 -> 资质审核 -> 线上调度 -> 进场核验 -> 安全交底 -> 使用监管 -> 退场评价”。
4. 使用正式论文语言，避免宣传稿口吻。
```

## 写系统设计章节

```text
请写“系统总体设计”章节。

要求：
1. 说明系统采用 ASP.NET Core MVC 分层架构。
2. 解释 Controller、Service、Entity、ViewModel、Razor View、AppDbContext 的职责。
3. 说明依赖注入、角色权限设计和状态驱动流程设计。
4. 不要把系统描述成微服务架构或前后端分离架构。
```

## 写数据库设计章节

```text
请写“数据库设计”章节。

要求：
1. 先说明数据库设计目标和原则。
2. 重点介绍 Equipment、Qualification、AuditRecord、DispatchRequest、DispatchOrder、Contract、EntryVerification、SafetyBriefing、InspectionRecord、FaultReport、ReturnApplication、ReturnEvaluation 等实体。
3. 说明以设备为核心、以调度单串联租赁过程的关系设计。
4. 说明主要状态枚举。
5. 不需要输出完整 SQL 建表语句。
```

## 写核心功能实现

```text
请写“系统详细设计与实现”章节中的核心功能实现部分。

要求：
1. 按用户认证与权限、设备台账与资质审核、线上调度与合同、进场核验、安全交底与使用监管、退场评价、首页看板与通知分节。
2. 每节按“功能目标 -> 实现流程 -> 涉及的 Controller/Service/Entity -> 状态变化”组织。
3. 语言要像论文，不要像代码注释。
4. 只写我提供资料中真实存在的功能。
```

