# 12 一致性规则

写论文或让 ChatGPT 改写时，必须遵守本文件，防止术语混乱和功能编造。

## 项目名称

统一使用：

```text
基于 ASP.NET Core MVC 的建筑租赁设备全生命周期管理平台
```

不要频繁改写为：

- 建筑机械租赁平台。
- 工程设备管理系统。
- 施工设备云平台。
- 智慧工地设备平台。

这些说法可以偶尔作为泛称，但论文题目、摘要、章节标题和核心描述应使用统一名称。

## 核心流程

统一写法：

```text
设备入库 -> 资质审核 -> 线上调度 -> 进场核验 -> 安全交底 -> 使用监管 -> 退场评价
```

不要改成：

- 采购入库。
- 在线审批。
- 现场扫码。
- 财务结算。

除非明确说明是未来扩展。

## 技术栈

统一写当前事实：

- C# 13。
- .NET 10。
- ASP.NET Core MVC。
- EF Core 10。
- SQL Server 2022。
- ASP.NET Core Identity。
- BCrypt。
- Razor Views。
- Bootstrap 5。
- jQuery 3。
- Chart.js 4。
- QuestPDF。
- EPPlus。
- QRCoder。
- HtmlSanitizer。

不要写成：

- Spring Boot。
- Vue、React 或 Angular。
- MySQL、PostgreSQL 或 Oracle。
- Redis、RabbitMQ、Elasticsearch。
- 微服务架构。
- 前后端分离架构。
- Docker Compose 部署架构。

## 角色名称

统一写：

- 系统管理员。
- 设备管理员。
- 调度员。
- 项目负责人。
- 安全员。
- 只读审计员。

注意：

- 只读审计员不是资质审核员。
- 资质审核操作由设备管理员或系统管理员执行。

## 状态名称

设备状态统一写：

- 待审核。
- 空闲。
- 出租中。
- 维修中。
- 已报废。

调度单状态统一写：

- 待签署。
- 已签署。
- 进行中。
- 已完成。
- 已终止。

合同状态统一写：

- 草稿。
- 待签署。
- 已签署。
- 已终止。

故障状态统一写：

- 待处理。
- 处理中。
- 已关闭。

## 已实现功能边界

可以写成已实现：

- 设备入库和台账管理。
- 设备图片上传。
- 证件管理和到期预警。
- 资质审核通过和驳回。
- 用车申请和调度排期。
- 合同在线预览、PDF 导出和扫描件上传。
- 核验码进场核验。
- 安全交底富文本、参与人签署和附件。
- 巡检记录和固定巡检项。
- 故障上报、处理和关闭。
- 退场申请、退场评价、押金扣款计算。
- 首页看板和站内消息。
- 文件上传安全校验。
- RBAC 权限控制。

只能写成未来展望：

- GPS 实时定位。
- 移动端 App。
- 真实硬件扫码设备。
- 第三方 ERP 集成。
- 财务发票系统。
- 在线电子签章法律认证。
- 多语言国际化。
- 分布式微服务改造。

## 论文语气

推荐表达：

- “系统采用……”
- “模块负责……”
- “服务层完成……”
- “通过状态字段描述……”
- “测试结果表明……”

避免表达：

- “本系统非常先进。”
- “彻底解决所有问题。”
- “达到企业级最高标准。”
- “完全自动化无需人工参与。”
