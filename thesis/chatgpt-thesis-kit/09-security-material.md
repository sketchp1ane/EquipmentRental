# 09 安全设计写作素材

## 认证与密码安全

系统使用 ASP.NET Core Identity 实现用户认证和角色管理，并配置登录失败锁定机制。密码使用 BCrypt 哈希存储，避免明文保存密码。

可写入论文：

系统在认证层采用 ASP.NET Core Identity 管理用户登录和角色信息，并结合 BCrypt 对密码进行哈希处理。即使数据库泄露，攻击者也难以直接获得用户明文密码，从而提升账号安全性。

## RBAC 权限控制

系统定义系统管理员、设备管理员、调度员、项目负责人、安全员和只读审计员六类角色。受保护页面和操作通过 `[Authorize]` 限定访问角色。

权限控制特点：

- 菜单按角色显示。
- Controller 和 Action 层执行授权。
- 只读审计员只能查看，不能审核或修改。
- 管理员拥有全局管理权限。

可写入论文：

系统采用基于角色的访问控制模型，将不同业务操作分配给不同角色。通过角色授权，系统能够限制用户只能访问与其职责相关的功能，减少越权操作风险。

## CSRF 防护

系统在 `Program.cs` 中全局注册 `AutoValidateAntiforgeryTokenAttribute`，对非 GET 请求自动校验防伪令牌。

可写入论文：

由于系统存在大量表单提交操作，系统启用全局 CSRF 防护机制，对修改类请求进行令牌校验，降低跨站请求伪造攻击风险。

## 文件上传安全

上传文件由 `FileService` 处理，文件存储在 `Uploads/` 目录，而不是 `wwwroot/`。访问文件时通过 `FilesController` 鉴权下发。

主要防护：

- 扩展名白名单，只允许图片和 PDF 等指定类型。
- 文件大小限制。
- MIME 类型校验。
- 文件魔数校验，防止伪造扩展名。
- GUID 重命名，避免重名覆盖和路径穿越。
- 上传文件不直接暴露在静态资源目录。

可写入论文：

系统对上传文件进行多层校验，包括扩展名、MIME 类型、文件头魔数和大小限制。文件保存时使用 GUID 重命名，并存放在非公开静态目录中，下载时统一经过控制器鉴权，从而降低恶意文件上传和未授权访问风险。

## 富文本安全

安全交底模块支持富文本内容。系统使用 HtmlSanitizer 在持久化前过滤危险 HTML。

可写入论文：

富文本输入可能包含恶意脚本，因此系统在保存安全交底内容前使用 HtmlSanitizer 进行过滤，防止危险标签或脚本进入数据库并在页面渲染时造成 XSS 风险。

## 审计与通知

系统通过操作日志和站内消息记录重要业务动作，例如审核、调度、交底、故障和退场等。

可写入论文：

审计日志和站内消息有助于记录关键业务节点，提升系统可追溯性和跨角色协同效率。

## 不应夸大的点

- 不要写系统通过硬件设备完成真实扫码。
- 不要写系统已经接入企业统一身份认证。
- 不要写系统具备金融级安全认证。
- 不要写系统实现了电子签章法律认证。
