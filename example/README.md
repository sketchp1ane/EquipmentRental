# 演示示例数据（example/）

本目录集中存放跑完整个业务流程所需的 **示例表单数据** 与 **占位图片**，方便直接复制粘贴与文件上传，无需临时拼凑内容。

> 数据库中 `DbInitializer` 已经 seed 了一条完整的演示链（DEMO-EQ-001~005）。
> 本目录里的数据是 **第二条链**（DEMO2-EQ-001/002），从设备入库一步步手动录入，用来演示 **端到端人工操作** 的完整过程。两条链互不冲突。

严格按编号执行即可，每一步的 md 文档都写明了：
- 使用哪个账号登录
- 进入哪个菜单路径
- 填写什么字段
- 上传哪张图片
- 下一步跳转到哪里

| 步骤 | 文档 | 角色 |
|---|---|---|
| 1 · 设备入库 + 证件 + 审核 | [`01-设备入库/设备信息.md`](01-设备入库/设备信息.md) | 设备管理员 |

| 3 · 合同预览 / 导出 / 扫描件 | [`03-合同签署/合同操作.md`](03-合同签署/合同操作.md) | 调度员 |

## 四、日期处理约定

md 里用 "演示当天 +N 天" 表示相对日期，录入时换算成实际日期即可。需要验证 "即将到期" 红色预警时，把该日期控制在 **距今 ≤30 天**。

## 五、重置演示数据

如需清空并重新跑一遍：

```bash
# 直接重建数据库容器
docker rm -f equiprental-db
docker run -e "ACCEPT_EULA=Y" -e "SA_PASSWORD=Admin123456!" \
  -p 1433:1433 --name equiprental-db -d \
  mcr.microsoft.com/mssql/server:2022-latest

# 重新应用迁移 + seed
dotnet ef database update
dotnet run
```

首次启动时 `DbInitializer` 会重新 seed 基础演示数据与所有演示账号。
